## Philosophy

Good developer documentation answers three questions before the reader asks them:

1. **What is this?** — A clear, one-sentence description of what the thing does.
2. **Why would I use it?** — The use case or problem it solves.
3. **How do I use it?** — The minimum viable steps to get a result.

Everything else is detail. Organize the detail so readers can skip to what they need.
